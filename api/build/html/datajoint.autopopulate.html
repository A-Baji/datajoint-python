datajoint.autopopulate module¶

autopopulate containing the dj.AutoPopulate class. See dj.AutoPopulate for more info.

class datajoint.autopopulate.AutoPopulate[source]¶

Bases: object

AutoPopulate is a mixin class that adds the method populate() to a Relation class. Auto-populated relations must inherit from both Relation and AutoPopulate, must define the property key_source, and must define the callback method _make_tuples.

key_source¶
Returns:the relation whose primary key values are passed, sequentially, to the _make_tuples method when populate() is called.The default value is the join of the parent relations. Users may override to change the granularity or the scope of populate() calls.
populate(*restrictions, *, suppress_errors=False, reserve_jobs=False, order='original')[source]¶

rel.populate() calls rel._make_tuples(key) for every primary key in self.key_source for which there is not already a tuple in rel.

Parameters:
  • restrictions – a list of restrictions each restrict (rel.key_source - target.proj())
  • suppress_errors – suppresses error if true
  • reserve_jobs – if true, reserves job to populate in asynchronous fashion
  • order – “original”|”reverse”|”random” - the order of execution
progress(*restrictions, *, display=True)[source]¶

report progress of populating this table :return: remaining, total – tuples to be populated

target¶

relation to be populated. Typically, AutoPopulate are mixed into a Relation object and the target is self.
